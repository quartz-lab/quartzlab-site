QuartzLab

Compact Unity Editor tools with one clear job each.

QuartzLab builds small Unity Editor plugins that cut repeated work without loading a project with frameworks it does not need. The goal is simple: save time, keep file changes predictable, and make the tool understandable the first time you open it.

Project links

GitHub YouTube Telegram

What QuartzLab is responsible for

QuartzLab focuses on practical Unity Editor workflows: importing, replacing, reviewing, and fixing assets without adding runtime systems or hidden pipelines. Tools stay editor-only unless there is a strong reason not to.

Core principles

One tool, one taskEach plugin should solve a specific problem and stay narrow enough to remain understandable.
Less routineAutomation should remove repeated manual work instead of replacing it with setup overhead.
Safe file operationsChanges should be explicit, reviewable, and reversible whenever possible.
Clear interfaceA tool should be usable without separate training material before basic work can begin.

How plugins are developed

QuartzLab targets Unity LTS first, keeps documentation in Russian and English, and prefers backwards compatibility when it can be maintained without hidden fragility. File writes should stay predictable, updates should stay understandable, and editor behavior should remain easier to audit than a custom framework.

Project history

QuartzLab is an independent one-developer project. It grew out of repeated production tasks during game development and editor tooling work, where small gaps in Unity workflows kept costing time. The project exists to package those fixes into focused plugins instead of carrying them as private snippets forever.

Licensing and source

Plugins are free and repositories stay open. Commercial use depends on the license declared in each repository and shown on the plugin page. The site now syncs that data from the latest published GitHub release so the version, minimum Unity version, license, and docs stay in one release source.
