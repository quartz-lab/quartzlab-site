QuartzLab

Компактные Unity Editor-инструменты, где у каждого одна понятная задача.

QuartzLab делает небольшие плагины для Unity Editor, которые сокращают повторяющуюся работу и не перегружают проект лишними системами. Цель простая: экономить время, держать изменения файлов предсказуемыми и делать инструмент понятным с первого запуска.

За что отвечает QuartzLab

QuartzLab сфокусирован на практических workflow для Unity Editor: импорт, замена, проверка и правка ассетов без добавления runtime-систем и скрытых пайплайнов. Если нет сильной причины иначе, инструменты остаются editor-only.

Основные принципы

Один инструмент — одна задачаПлагин должен решать конкретную проблему и оставаться достаточно узким, чтобы его было легко понять.
Меньше рутиныАвтоматизация должна убирать повторяющиеся действия, а не заменять их новой прослойкой настроек.
Безопасные операции с файламиИзменения должны быть явными, проверяемыми и по возможности обратимыми.
Понятный интерфейсИнструмент должен быть полезен без отдельного обучения перед первым рабочим сценарием.

Подход к разработке

QuartzLab в первую очередь ориентируется на Unity LTS, поддерживает документацию на русском и английском и по возможности сохраняет обратную совместимость без скрытой хрупкости. Изменения файлов должны быть предсказуемыми, обновления — понятными, а поведение редактора — проще для аудита, чем самодельный фреймворк.

История проекта

QuartzLab — независимый проект одного разработчика. Он вырос из повторяющихся задач в разработке игр и работе с editor tooling, когда небольшие пробелы в workflow Unity постоянно съедали время. Проект нужен для того, чтобы превращать такие решения в отдельные сфокусированные плагины, а не хранить их бесконечно в личных заготовках.

Лицензирование и открытый код

Плагины бесплатные, а репозитории остаются открытыми. Использование в коммерческих играх зависит от лицензии, указанной в репозитории и показанной на странице плагина. Сайт теперь синхронизирует версию, минимальную Unity, лицензию и документацию из последнего опубликованного GitHub-релиза, чтобы источник релизных данных был один.
