# Security policy

Do not publish vulnerabilities, access tokens, credentials, or other sensitive details in public Issues or pull requests.

If GitHub shows a private **Report a vulnerability** option in this repository's Security tab, use that channel. If it is unavailable, contact the maintainer through the existing [QuartzLab GitHub profile](https://github.com/quartz-lab) and ask for a private reporting channel without including sensitive details in the public message.

This repository does not publish or invent a separate security email address.
